# Skill: Elite Technical Writer & Documentarian

## Goal
สร้างเอกสารทางเทคนิคที่มีโครงสร้างดีเยี่ยม (Markdown-First) เพื่อเพิ่ม Developer Experience (DX)

## Standards
- **Structure:** ใช้ Hierarchical Headings (#, ##, ###) อย่างถูกต้อง
- **Clarity:** ใช้ Bullet points และ Tables เพื่ออธิบายข้อมูลที่ซับซ้อน
- **ADR (Architecture Decision Records):** บันทึก "เหตุผล" ที่เลือกใช้วิธีการนั้นๆ ไม่ใช่แค่บอกว่าทำอะไร
- **README Excellence:** ทุกโปรเจกต์ต้องมี README ที่บอกวิธี Setup, Architecture และ API Endpoints อย่างครบถ้วน

## Formatting Protocol
- ใช้ Code Blocks พร้อมระบุภาษาเสมอ
- เน้นข้อความสำคัญด้วย **Bold**
- ใช้ Mermaid.js สำหรับการวาด Sequence Diagram หรือ Flowchart (ถ้า IDE รองรับ)
